Data Cleaning

Data cleaning involves identifying and handling missing or inconsistent data, removing duplicates, and transforming the data to a format suitable for analysis or machine learning.

Feature Engineering

Feature engineering involves selecting and transforming raw data into a new set of features that can better represent the underlying problem to improve the performance of machine learning models.

Exploratory Data Analysis

Exploratory Data Analysis (EDA) involves analyzing and summarizing the main characteristics of a dataset through visual and quantitative methods to better understand the patterns, relationships, and distributions of the variables before applying any modeling techniques.

Train-Test Split

Train-test split involves dividing the original dataset into two subsets, one for training the model and the other for evaluating its performance on unseen data, using a specific ratio or proportion, such as 80:20 or 70:30, respectively, to prevent overfitting and improve generalization.

SMOTE

SMOTE (Synthetic Minority Over-sampling Technique) is a popular data augmentation technique used in imbalanced machine learning tasks that generates synthetic examples of the minority class by interpolating between neighboring instances in the feature space.

Splitting the data into Numeric and Categorical

Splitting the data into numeric and categorical involves identifying and separating the variables in a dataset that are continuous and can be represented by numerical values from those that are discrete and categorical in nature.

Standardization of Numeric DataFrame

Standardization of a numeric DataFrame involves transforming the values of the continuous variables so that they have a mean of zero and a standard deviation of one, making them more comparable and easier to interpret when applying machine learning algorithms.

10

Encoding of Categorical DataFrame

Categorical encoding involves transforming categorical variables into numerical or binary representations that can be used as input for machine learning models, such as one-hot encoding, ordinal encoding, or target encoding.

11

Combining Categorical & Numerical DataFrames

Combining categorical and numerical DataFrames involves merging the two data frames into a single data frame using a common identifier or index, which can then be used for further analysis or modeling tasks.

12

Model Building

Model building involves selecting a suitable algorithm, training the model on the dataset, evaluating its performance on a validation set, and fine-tuning the model's hyperparameters to achieve the best possible accuracy or performance for a given task.

Evaluating the Model

Evaluating a model involves using metrics such as accuracy, precision, recall, F1-score, ROC-AUC, or others, to assess the model's performance on the test set, and compare it against other models or baselines, to determine its effectiveness and generalization capability.

14

Predicting using the Model

Predicting using the model involves feeding new data into the trained model to obtain predictions or classifications based on the learned patterns and relationships from the training data, which can be used for making decisions or generating insights for a specific problem or application.
